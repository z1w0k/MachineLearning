## Описание проекта
Разработка рекомендательной системы на Python, где нейросеть анализирует историю просмотров пользователя и предсказывает, какой фильм он захочет включить следующим. Главная фишка проекта — использовать архитектуру трансформеров (SASRec) для понимания хронологического порядка действий, корректно сплитовать данные по времени без «подглядывания в будущее» и оценивать точность по реальным бизнес-метрикам ранжирования (попадание в топ-10).

## Стек технологий
* **Язык:** Python
* **Библиотеки:** PyTorch (Transformer / nn.Embedding), Pandas
* **Домен:** RecSys, SASRec, ML Metrics (HR@K, NDCG@K)

---

## Архитектура и особенности
1. **Кастомный Dataset:** Написана правильная обработка временных последовательностей с левосторонним паддингом (`padding_idx=0`) и сдвигом таргетов на 1 шаг вперед.
2. **Causal Masking:** Внутри трансформера используется каузальная маска, которая закрывает от модели "будущие" шаги при обучении на истории.
3. **Time-aware Split:** Данные разбиваются по принципу *Leave-One-Out* (последний фильм — в тест, предпоследний — в валидацию, всё остальное — в трейн). Никакого случайного перемешивания, чтобы избежать утечки данных (*data leakage*).
4. **Бизнес-метрики:** Модель оценивается не по лоссу, а по метрикам ранжирования **Hit Rate@10** и **NDCG@10**. Результат работы бейзлайна превышает случайное угадывание **в 8 раз**.

## Структура кода
Весь проект реализован в одном понятном файле:
* `split_sequential_data()` — логика временного разбиения на Train/Val/Test.
* `SequentialRecDataset` — класс подготовки данных и батчинга под PyTorch.
* `SASRec` — нейросетевая архитектура на базе `nn.TransformerEncoder`.
* `evaluate_model()` — расчет офлайн-матриц HR@10 и NDCG@10 на валидации и тесте.
* `train_sasrec()` — цикл обучения с оптимизатором Adam и CrossEntropyLoss.
* `main()` — автоматическая загрузка датасета **MovieLens 100K** и запуск пайплайна.
